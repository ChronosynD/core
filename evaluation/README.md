# Evaluation

The *data* side of evaluation lives here: workload fixtures, attack-payload definitions, and the CSV results experiments write here. The *code* side lives inside the Python package at [`../chronosynd-py/chronosynd_py/evaluation/`](../chronosynd-py/chronosynd_py/evaluation/) so pytest, ruff, and mypy cover it under the same quality gates as the rest of the library.

## Layout

| Path | Purpose |
|---|---|
| `workloads/` | Synthetic workload factories live in code; captured JSONL traces from `chronosynd run --record` land here. See [`workloads/README.md`](workloads/README.md). |
| `attack_payloads/` | Behavioral fixtures used by the real-data demo. See [`attack_payloads/README.md`](attack_payloads/README.md). |
| `results/` | Output CSVs written by experiments, regenerated by `bash ../scripts/reproduce_all.sh`. |

## Threat model

See [`../docs/THREAT_MODEL.md`](../docs/THREAT_MODEL.md) for the formal adversary definition. The attacks under `chronosynd_py.evaluation.attacks` are concrete instantiations of that model.

## Experiments

| Experiment | Question it answers |
|---|---|
| `exp01_clean_baseline_fpr.py` | Under unpoisoned windows, every estimator's FPR lands at the configured target by construction. Sanity check, not a differentiation panel. |
| `exp02_naive_under_poisoning.py` | **Motivation figure.** How sharply does the naive estimator's target score collapse as the adversary's budget grows? |
| `exp03_robust_under_poisoning.py` | **Main result.** How does the budget × trim_fraction design space play out for Sediment on isotropic benign? |
| `exp04_budget_sweep.py` | Naive versus Sediment overlaid across budgets, with three views of the same data: target score, threshold-invariant score ratio, detection rate at target FPR. |
| `exp05_fp_cost.py` | What is the clean-condition FPR of Sediment at increasing trim fractions? |
| `exp06_multimodal_benign.py` | Sediment on a bimodal benign distribution. Exposes the structural limit of independent-Gaussian baselines that the discussion section addresses honestly. |
| `exp07_alternatives_ablation.py` | **Four-way ablation, uniform poisoning.** Naive vs Consensus vs AnomalyWithin vs Sediment under `pre_seed`. |
| `exp08_burst_alternatives_ablation.py` | **Four-way ablation, burst poisoning.** Same four estimators under contiguous-block `inject_burst`. Surfaces that simple consensus filtering is not the right primitive against this threat model. |
| `exp09_targeted_white_box.py` | **White-box adversary.** Same four estimators under `inject_targeted`, the grid-search adversary that knows Sediment's `trim_fraction`. The strongest threat-model evidence the paper produces. |

## Reproduction

To regenerate every CSV here and every PDF in [`../paper/figures/`](../paper/figures/) from scratch:

```bash
bash ../scripts/reproduce_all.sh
```

The full pipeline runs in under two minutes. To run the experiment:

```bash
cd ../chronosynd-py
uv run python -m chronosynd_py.evaluation.experiments.exp04_budget_sweep
uv run python -m chronosynd_py.evaluation.figures.fig04_budget_sweep
```

CSVs land in `results/`. PDFs land in `../paper/figures/`.

## Real-data evaluation

The synthetic experiments above are the paper's controlled comparisons. To run the detector against real captured behavior:

```bash
# Build the BPF-enabled collector and CLI
cd ../chronosynd-rs && cargo build --release --features bpf

# Capture, fit, attack, and score in one driver
sudo ../scripts/real_world_demo.sh nginx 600 ../evaluation/attack_payloads/etc_recon.sh
```

The real-data protocol is documented in [`../docs/evaluation_protocol.md`](../docs/evaluation_protocol.md).
